**사용자가 사업 아이디어를 입력하면,  
시장성·경쟁·수익성·리스크·실행 난이도 등을 자동 분석해주는 AI 에이전트.**
### 🎯 1. 해결하려는 문제(Pain Point)

사람들은 사업 아이디어가 떠올라도 이런 문제를 겪어:

- 시장조사를 어떻게 해야 할지 모름
- 경쟁 분석을 할 능력이나 시간 부족
- 수익 모델을 어떻게 검증해야 할지 모름
- 사업 타당성 보고서를 만들기 어려움
- 전문가 컨설팅은 비싸고 느림

즉, **“사업 아이디어 검증의 진입장벽”**이 높아.

이 에이전트는 이걸 해결해.

### 🎯 2. 타겟 고객

이 아이디어는 타겟이 꽤 넓어.

- 예비 창업자
- 스타트업 초기 팀
- 프리랜서/1인 기업
- 대학생 창업 동아리
- 기업 내 신사업 담당자
- VC/엑셀러레이터의 초기 스크리닝 도구

특히 **예비 창업자 시장은 크고, 정보 비대칭이 심해서** 니즈가 강해.

### 🎯 3. 경쟁/대체재

현재 사람들이 쓰는 대체재는:

- 유튜브 검색
- 블로그/카페 정보
- 창업센터 컨설팅
- 시장조사 리포트
- ChatGPT 같은 범용 AI

하지만 **전문적으로 사업 타당성만 분석하는 AI 서비스는 거의 없음**.  
즉, **니치(niche) + 성장 가능성**이 있는 시장.

### 🎯 4. 수익 모델

여러 모델이 가능해.

✔ 구독 모델

- 월 9,900원~29,000원
- 무제한 아이디어 분석

✔ 건당 분석 모델

- 아이디어 1개 분석 = 3,000~10,000원

✔ B2B 모델

- 스타트업 센터, 엑셀러레이터, 대학 창업센터에 SaaS 형태로 판매

✔ 프리미엄 리포트

- 자동 생성된 사업 타당성 보고서 PDF 판매

수익화 구조가 꽤 명확해.

### 🎯 5. 실행 난이도

기술적으로는 난이도가 중간 정도.

- AI 모델 자체는 존재
- 핵심은 **프레임워크 설계 + 데이터 구조화 + UX**
- 시장·경쟁·수익성 분석을 자동화하려면  
    “정형화된 질문 체계 + 논리적 평가 구조”가 필요

즉, **기술보다 기획력이 더 중요한 서비스**.

### 🎯 6. 리스크

이 아이디어의 가장 큰 리스크는 다음 두 가지.

1) **데이터 기반 분석의 한계**

AI는 실제 시장조사나 고객 인터뷰를 대체할 수 없음.  
→ “너무 AI만 믿고 창업했다가 실패” 같은 문제가 생길 수 있음.

2) **책임 문제**

AI가 잘못된 분석을 했을 때 책임 소재가 모호함.

3) **차별화 문제**

범용 AI(예: 나 같은 Copilot)와의 차별화를 어떻게 만들지.

### 🎯 7. 차별화 포인트(USP)

이 서비스가 성공하려면 아래가 핵심 USP가 돼야 해.

- **사업 타당성 분석에 특화된 구조화된 질문 체계**
- **시장 규모 추정 자동화**
- **경쟁사 맵핑 자동화**
- **리스크 가설 도출 기능**
- **실행 로드맵 자동 생성**
- **사업계획서 자동 생성 기능**

즉, “범용 AI가 아닌, 사업 분석 특화 AI”라는 포지션.

### 🎯 8. 결론: 사업성 평가

이 아이디어는 **충분히 사업성이 있다**고 볼 수 있어.

- 시장 니즈 존재
- 경쟁 적음
- 기술적 진입장벽 중간
- 수익 모델 명확
- 확장성 높음
- B2C + B2B 모두 가능

단, **AI가 실제 시장조사를 대체할 수 없다는 한계**를 어떻게 보완할지가 핵심.

## 수집할 데이터 종류
### 내부 데이터 (사용자가 입력한 정보)

이건 “아이디어 자체의 속성”을 평가하는 데 쓰여.

예:

- 해결하려는 문제 정의
- 타겟 고객
- 제공 가치
- 수익 모델
- 경쟁 우위
- 실행 가능성
- 창업자의 역량

이건 외부 데이터로는 절대 알 수 없고,  
**사용자가 제공해야만 하는 정보**야.

내부 데이터는 “아이디어의 논리적 타당성”을 평가하는 데 쓰임.

### 외부 데이터 (시장·경쟁·트렌드·수요 관련 정보)

이건 “아이디어가 시장에서 먹히는지”를 판단하는 데 쓰여.

예:

- 시장 규모
- 성장률
- 경쟁사 수
- 대체재
- 소비자 행동 패턴
- 가격 민감도
- 규제
- 기술 트렌드

즉, 외부 데이터는 **아이디어의 시장 타당성**을 평가하는 데 쓰임.

## 외부 데이터 사용 범위
너무 넓으면:

- 분석이 느려지고
- 정확도 떨어지고
- 비용 증가

너무 좁으면:

- 중요한 리스크 놓침
- 시장성 판단 오류
따라서 
✔ 산업 규모: 3~5개 주요 출처

✔ 경쟁사: 상위 5~10개

✔ 고객 데이터: 검색량·리뷰·SNS 정도

✔ 규제: 정부·지자체 공식 문서

✔ 기술 트렌드: 최근 1~2년 자료

✔ 수익성: 업종 평균 벤치마크
이 정도 범위가 적당함. 이 정도면 **실제 컨설팅 회사 수준의 분석**이 가능.

## 외부 데이터 분류
실제로 사업 타당성 평가에 필요한 외부 데이터는 **6가지 범주**로 정리할 수 있어.

### 📊 **① 시장 규모 & 성장률 (TAM/SAM/SOM)**

- 산업 보고서
- 정부 통계
- 시장조사 기관 자료
- 검색 트렌드
- 소비자 수요 지표

이건 “이 시장이 돈이 되는가”를 판단하는 핵심.

### 🧩 **② 경쟁사 & 대체재 분석**

- 주요 경쟁사
- 가격대
- 차별화 포인트
- 시장 점유율
- 고객 리뷰 분석

이건 “경쟁이 얼마나 빡센가”를 판단하는 핵심.

### 🧍 **③ 고객 행동 데이터**

- 검색량
- 구매 패턴
- 리뷰 패턴
- SNS 언급량
- 고객 페인포인트

이건 “고객이 실제로 원하는가”를 판단하는 핵심.

### ⚖️ **④ 규제·법률·인허가 정보**

- 정부 규제
- 인증 필요 여부
- 지역 제한
- 세금 구조

이건 “실행 가능성”을 판단하는 핵심.

### 🛠 **⑤ 기술 트렌드 & 산업 동향**

- 최신 기술 변화
- 산업 구조 변화
- 투자 트렌드
- 특허 동향

이건 “미래 지속 가능성”을 판단하는 핵심.

### 💰 **⑥ 수익성 벤치마크**

- 업종별 평균 마진
- CAC/LTV 벤치마크
- 인건비/원가 구조
- 초기 투자 규모

이건 “돈이 실제로 남는가”를 판단하는 핵심.


## **AI 에이전트가 실제로 할 수 있는 일**

AI는 위 6가지 외부 데이터를 기반으로 다음을 자동화할 수 있어:

- 시장 규모 추정
- 경쟁사 맵핑
- 고객 페인포인트 추출
- 가격대 분석
- 수익성 모델링
- 리스크 가설 도출
- 실행 로드맵 생성

즉, **AI는 외부 데이터를 기반으로 “정량적 판단”을 하고,  
내부 데이터를 기반으로 “논리적 판단”을 한다.**
